Research Projects

Wi-Fi LTE-U Interference (Sept 2014 - May 2015)

This project was to study the interference to Wi-Fi caused by LTE-U coming into unlicensed bandwidths and to design a coordination system in which LTE-U and Wi-Fi can coexist peacefully. My role in this project included FPGA programming, some driver programming in C, experiment design, experiment execution and data analysis of the results compared to a theoretical model that we constructed. This required C, Ruby and MATLAB programming, LTE emulation software called OpenAirInterface, in-depth knowledge of how Wi-Fi and LTE-U operate (including Markov chain analysis and packet scheduling).

See publication section for results and details.

Directional Networking Antenna (Sept 2015 - Present)

The project is to design an OFDM-based waveform for use with a directional networking antenna system. My role in this project was to assist in developing the proposal and is currently to develop a test structure for the receivers. My test structure allows the user to easily program a stream of commands (one byte each, numbers 1 - 64) using an LCD shield with buttons and an arduino. The arduino then processes the user input and sends either a RAW or SLIP packet through RS485 serial outward towards the receiver. This requires C/C++, arduino, raspberry pi and a little bit of circuit knowledge to accomplish.

3D Mapping and Collision Avoidance (Sept 2015 - Present)

This project is for a UAV known as the "Naviator" to achieve collision avoidance and 3D mapping. We started with a LiDAR and some cheap MEMS gyros to run some tests. Once we concluded those tests, we moved on to using a Stereolabs ZED 3D camera to generate point clouds for this purpose. With this camera, I have used the C++ Point Cloud Library to do point cloud stitching and use the point clouds to perform computations. Eventually, this will be moved on to an ARM system called the Jetson TX1 where it can be mounted on the Naviator and used in real-time. This requires Python, C++, Point Cloud Library, Boost C++ Library, MATLAB, arduino, low-level IO, computer vision and linear algebra knowledge to accomplish.

Telephone Pole Sensor Network (Jan 2016 - Present)

This project is to develop a sensor network to predict when a telephone pole is most likely to fall and to notify a base station whenever a pole is predicted to fall or has already fallen. Our design involves using weather detection and proximity sensors along with resistance analysis of the wood to determine which pole is most likely to fall from external anomalies. To determine whether a pole has already fallen, we use a gyro to pick up a tilt below a certain level and to set off an alarm. To communicate this information back to the base station, we use a mesh network because of its resilience to network damage and missing nodes in the network. Our system is powered by induction from the power lines. Our system is very resilient to outside damage and node outages and is self-sustaining and self-healing. We are implementing this sensor network using circuit design for the induction power harvesting and storage and raspberry pi zeros to do all of the computations and networking. This project requries C++, Python, IEEE 802.15.4 protocol knowledge, low-level GPIO and I2C, machine learning and systems engineering to accomplish.
